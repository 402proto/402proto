# 402PIP-01: envelope shape

**status:** accepted (v0.1)
**author:** 402proto
**created:** 2026-04-15

## summary

defines the http 402 envelope used by all clients and providers in 402proto. the envelope is a small set of `x-402proto-*` headers exchanged between a client, an optional router, and a provider.

## motivation

paid api calls need a portable shape that:

- works over plain http (no custom protocol layer)
- carries enough information for an on-chain proof of payment to be derived after the call settles
- avoids leaking provider-specific structure into the wire format

an `x-402proto-*` header set lets any http server emit the envelope by adding response headers, and any http client read it by inspecting status + headers. no extra body parsing required for the protocol layer.

## proposal

see [spec/envelope.md](../envelope.md). the proposal text was merged verbatim into that file when this PIP was accepted.

key shape:

- **request side**: `x-402proto-version`, `x-402proto-client`, `x-402proto-wallet`, optional `x-402proto-max-price`, optional `x-402proto-idempotency-key`
- **response 402**: provider, method, price, currency, recipient, ttl, chain, quote-id, nonce
- **response 200**: payload + `x-402proto-receipt` echo

## rationale

- headers instead of body — providers can adopt the protocol without changing their existing response shape.
- version header first — lets routers reject mismatched versions cheaply.
- explicit `quote-id` + `nonce` — defeats replay attacks; nonce is part of the canonical body signed by the wallet.
- ttl in seconds — kept small (default 12) to limit window for quote front-running.

## acceptance

merged into `spec/envelope.md` as v0.1. all v0.1 clients and providers MUST conform.
