# 402PIP-03: usdc spl-transfer settlement on solana

**status:** accepted (v0.1)
**author:** 402proto
**created:** 2026-04-16

## summary

v0.1 settles paid calls via a plain spl-token transfer of usdc on solana, with a memo program instruction tagging the transfer with the quote-id. providers verify by querying the wallet's recent transactions and matching memo + amount + recipient.

## motivation

we need a settlement primitive that is:

- native to solana (no bridge, no wrapped tokens, no custom contract)
- verifiable by the provider with one rpc call
- traceable on-chain forever (audit trail, billing reports, no off-chain log)

a stock spl-token transfer with a memo satisfies all three. no contract to audit, no upgrade path to fight about.

## proposal

see [spec/settle.md](../settle.md). key details:

- usdc mint: `EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v` (mainnet) / `Gh9ZwEmdLJ8DscKNTkTqPbNwLNNBjuSzaG9Vp2KGtKJr` (devnet)
- memo program: `MemoSq4gqABAXKb96qnH8TysNcWxMyWCqXgDLGmfcHr`
- memo content: `402proto/0.1 q=<quote-id>`
- amount: `x-402proto-price` × 10^6 (usdc has 6 decimals)
- provider waits for status `Confirmed` or `Finalized` before serving payload

## rationale

- **direct transfer (not contract)**: zero on-chain code to audit. v0.2 introduces an optional allowance contract for fleets that want to skip the per-call signing overhead, but v0.1 keeps the rail boring on purpose.
- **memo for quote-id**: the only way to bind an off-chain quote to an on-chain transfer without a custom program. memos are cheap (~$0.000005 in extra fees).
- **ttl + nonce in signed body**: prevents an attacker from replaying an old quote's settle path even if they capture the signed payload.
- **provider verifies the chain, not the client**: chain is the source of truth. clients can lie about txids; providers MUST query rpc.

## what is NOT in v0.1

- **no escrow / allowance**: every call is a fresh signed transfer. fleets pushing 1m+ calls/day will want allowance flow, scheduled for v0.2 (see 402PIP-04).
- **no native usdc on l2**: solana only. v0.4 adds eip-3009 settlement on base/arb/op (see 402PIP-06).

## acceptance

merged into `spec/settle.md` as v0.1. mainnet contract is not deployed yet — devnet only — but the wire format and verification rules are frozen.
